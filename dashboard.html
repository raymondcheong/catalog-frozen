期貨 · 航運綁定

操作檢查清單（實現：後台先綁定 → 客戶點連結 → 第三方頁帶單號）

  1. 在 products.js 為該 SKU 設 logistics: { live: true }，並部署。
  2. Netlify（或本機 .env）設定 SHIPPING_OPS_SECRET，與下方密碼一致。
  3. 無付費 API 時建議 SHIPPING_PROVIDER=public；有 SeaRates 金鑰則設 searates 並填 SEARATES_API_KEY。
  4. 下表選產品 → 單證模式 → 必填：追蹤類型（CT/BK/BL）、追蹤單號、船公司（勿選「自動」才可把「【優先】官網…」連結置頂；達飛選 CMDU）。
  5. 選填：船東顯示名、ETA、裝/卸港、地圖座標、船名航次（讓目錄頁更美，且不影響連結預填）。
  6. 點保存綁定；到產品目錄打開詳情，確認「公開查詢」連結；另開 Track-Trace 應見單號已插入。

方式 A · 單證綁定：填 櫃號／訂艙／提單 與船公司。若未設定 SeaRates API，目錄頁會顯示您綁定的真實單證並附加 track-trace／部分船公司官網公開查詢連結（連結網址內預先帶好單號，客戶另開分頁查即時狀態）；若已設定 SEARATES_API_KEY 則由後端自動查船期。
方式 B · 手動維護：僅有船名／航次或紙面 ETA 時，直接填狀態、預計到港、目的港等；客戶打開產品即可看到，無需輸入單證。 （行業 API 多數無法「只輸船名航次」自動出軌跡，故以後台維護為準。）
環境變數：SHIPPING_OPS_SECRET、 SHIPPING_PROVIDER 建議 public（預設，無金鑰亦可）或 searates（需 SEARATES_API_KEY）；mock 僅作內部假資料演示。手動模式不依賴 SeaRates。
用瀏覽器網址打開卻沒表單？代表線上站尚未部署最新檔案；請改從資料夾雙擊本機 dashboard.html，或先部署再開網址。

須與 Netlify 環境變數 SHIPPING_OPS_SECRET 完全相同（本機可另在專案 .env 設定供 Function 讀取）。密鑰只可寫在 Netlify／本機環境，勿提交到 Git。
