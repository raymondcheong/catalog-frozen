# 數據監控部署教學（新手版）

這份教學會一步步帶你完成「用 deploy.bat 部署」和「連結到正確的 Netlify 站點」。

---

## 一定要用 Git 嗎？

**不用。** 使用 `deploy.bat` 即可完成部署，無需 Git。Netlify 支援多種部署方式：
- **deploy.bat（CLI）**：你目前使用的方式，可部署靜態檔案 + Functions
- Git 連續部署：適合有程式版本控管需求時使用
- 手動 ZIP 上傳：僅靜態檔案，無法部署 Functions

---

## 事前準備

### 1. 確認專案位置

你的專案在：
```
C:\Users\admin\Desktop\CURSOR\catalog-frozen
```

請用**檔案總管**打開這個資料夾，確認裡面有：
- `deploy.bat`（部署腳本）
- `index.html`（產品目錄）
- `dashboard.html`（數據監控）
- `netlify` 資料夾（裡面有 functions）
- `package.json`

### 2. 安裝 Node.js（若尚未安裝）

1. 開啟瀏覽器，前往 https://nodejs.org/
2. 下載並安裝 **LTS 版本**
3. 安裝完成後，關閉所有終端機視窗，再重新開啟

## 第一步：執行 deploy.bat

### 做法

1. 開啟**檔案總管**
2. 前往 `C:\Users\admin\Desktop\CURSOR\catalog-frozen`
3. 找到 **deploy.bat**
4. **雙擊** `deploy.bat`（用滑鼠左鍵點兩下）

### 會發生什麼事

- 會跳出一個**黑色視窗**（命令提示字元）
- 視窗裡會顯示「複製專案到 C:\temp...」「安裝依賴...」等訊息
- 請**不要關閉**這個視窗，等它跑完

### 若出現「請以系統管理員身分執行」

- 在 `deploy.bat` 上按**右鍵**
- 選擇「以系統管理員身分執行」

---

## 第二步：連結到 Netlify 站點

執行 deploy.bat 後，黑色視窗可能會出現**選項**，請依序操作：

### 情況 A：出現「What would you like to do?」

畫面會像這樣：
```
? What would you like to do? (Use arrow keys)
> Link this directory to an existing site
  + Create & configure a new site
```

**操作方式：**
1. 用鍵盤的 **↑** 或 **↓** 方向鍵，讓「Link this directory to an existing site」前面有 `>` 符號
2. 按 **Enter** 確認

### 情況 B：出現「Select a site」

會列出你的 Netlify 站點，例如：
```
? Select a site:
> famous-manatee-f83275
  thriving-jalebi-842c2e
  (其他站點...)
```

**操作方式：**
1. 用 **↑** 或 **↓** 方向鍵，選到 **famous-manatee-f83275**（或你要用的產品目錄站點）
2. 按 **Enter** 確認

### 情況 C：出現「Log in to Netlify」

若尚未登入 Netlify：
1. 會自動開啟瀏覽器
2. 在瀏覽器裡登入你的 Netlify 帳號
3. 登入成功後，回到黑色視窗繼續操作

### 情況 D：沒有出現選項，直接開始部署

表示之前已經連結過，會直接開始上傳，等待完成即可。

## 第三步：等待部署完成

部署時會顯示類似：
```
✔ Finished uploading blobs to deploy store
✔ Hashing files...
✔ Deploying...
```

看到 **「Deploy complete!」** 或 **「部署完成！」** 就代表成功了。

## 常見問題

### Q1：黑色視窗閃一下就關了

- 可能是 Node.js 未安裝或路徑有問題
- 請在 `catalog-frozen` 資料夾裡，按住 **Shift** 再按**右鍵**，選「在此處開啟 PowerShell 視窗」
- 輸入 `node -v` 按 Enter，若有顯示版本號表示 Node.js 正常

### Q2：出現 EBUSY 錯誤

- 請再執行一次 `deploy.bat`
- 腳本會自動處理，若仍失敗，請手動刪除 `C:\Users\admin\.netlify` 資料夾後重試

### Q3：找不到 famous-manatee-f83275

- 可能你的 Netlify 帳號裡沒有這個站點
- 請選你實際有在用的產品目錄站點（例如 thriving-jalebi-842c2e）
- 部署完成後，用該站點的網址 + `/dashboard.html` 開啟數據監控

### Q4：部署成功但 dashboard 仍顯示「無法連線」

- 可能是用 ZIP 手動上傳的，沒有包含 Functions
- 請改用 `deploy.bat` 部署一次
- 若 deploy.bat 已成功，請到 Netlify 後台 → 站點 → **Deploys** → 點最新一筆部署 → 確認是否有顯示「Functions deployed」
- 若站點有連接 Git 自動部署，Git 的部署可能會覆蓋 CLI 部署；請確保 deploy.bat 部署的是你要用的站點（famous-manatee-f83275）
